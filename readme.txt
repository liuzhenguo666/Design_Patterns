基于KFC对建造者模式与工厂方法模式实现的简单说明
1.建造者模式
    使用Java实现时参考提供的源码，分为套餐基类，建造者基类，套餐1建造者类，套餐2建造者类，搭配服务员类。使用最简单的方法检测程序正确性。
    建造者模式使各个部分分工明确，各司其职。
2.工厂方法模式
    在建造者模式的基础上，按抽象工厂，具体工厂，抽象产品，具体产品的设计思想分别编程实现。在测试程序中检测基本功能。
    工厂方法模式对拓展开发，对修改封闭的开闭原则。
3.建造者模式与工厂方法模式的结合实现
    (1)在建造者模式与工厂方法模式单独实现时都忽略了套餐中的主食和饮料分别可以出现多个不同的套餐中。
    (2)在建造者模式中，套餐n建造者类身兼建造主食和饮料，比较复杂臃肿，不利于后期维护。
    (3)在工厂方法模式中，总体结构比较复杂，重用性低，使程序的阅读性降低。
    (4)对于以上三点的思考，在建造者模式与工厂方法模式结合实现中，我的设计思想是将套餐中的各种主食和饮料分别使用工厂方法模式实现，再使用建造者模式实现各个套餐的搭配组装。这样实现有利于后期添加主食或饮料，和使用主食与饮料不同的组合的套餐。
